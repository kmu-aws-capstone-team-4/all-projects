# PM Sub Agents 운영 가이드

## 목적
각 PM(프로젝트 매니저)이 Sub Agent(하위 에이전트)와 상호 통신하며 업무를 분담하도록 표준 절차를 제공합니다.
PM은 기획과 조율에 집중하고, 실행은 하위 에이전트에게 위임합니다.

## 역할 정의

### PM 역할
- 목표, 범위, 우선순위 정의
- 하위 에이전트에게 업무 할당 및 의사 결정 정리
- 결과 통합 및 이해관계자 보고

### Sub Agent 역할
- 할당된 실행 작업 수행
- 진행 상황과 결과 요약 제공
- 리스크 및 차단 요인 보고

## 운영 원칙
- 단일 PM이 전체 업무를 통합 책임
- 작업은 “목표 → 작업 단위 → 검증” 흐름으로 분해
- 모든 결과물은 요약 보고로 수집
- 변경 범위가 큰 경우, 단계별로 분리하여 요청

## 표준 작업 절차

1. 요구사항 정리
   - PM이 목표, 범위, 완료 기준을 정의
2. 하위 에이전트 배정
   - 실행 역할에 따라 최소 1명 이상 배정
3. 작업 요청
   - 작업 범위, 입력, 출력 형식을 명시
4. 진행 확인
   - 중간 산출물 확인 및 피드백 제공
5. 결과 통합
   - 결과를 통합하고 리스크를 정리
6. 검증 요청
   - 테스트 또는 도메인 검증 요청

## 요청서 기본 형식

- 목표: 무엇을 달성해야 하는지
- 범위: 포함/제외 범위
- 입력: 참조 문서, 코드, 데이터
- 출력: 기대 결과 형식(문서/코드/테스트 등)
- 제약: 일정, 품질, 정책

## 보고 형식

- 진행 요약: 현재 상태
- 변경 사항: 핵심 변경점
- 리스크: 발견된 문제와 대응
- 다음 행동: 후속 작업 제안

## 예외 처리

- 차단 요인 발생 시 즉시 보고
- 권한 부족, 경로 불일치 등은 PM에게 재확인 요청
